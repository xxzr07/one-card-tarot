# Deck 01 — Quiet Geometry

## 1. Core concept

Rider–Waite–Smithの主要な象徴を読み取れる範囲で残しながら、人物中心の場面を「余白・幾何学・光と影・自然物・痕跡」へ置き換える。カードの意味を説明しすぎず、象徴同士の距離と配置によって静かな解釈の余地をつくる。

## 2. Visual rules

- 縦長比率：612 × 1206 px（約 1:1.97）
- 外周：角丸。外縁から一定距離に細い一重フレーム
- 番号：上部中央。ローマ数字を基本とし、THE FOOLのみ `0`
- 英語名：下部中央。字間を広く取る
- 補助記号：カード名の下に小さく配置。意味の主役にはしない
- 線：極細を基本とし、輪郭をすべて描かず余白へ消す
- 質感：均一なフラット塗りではなく、紙と鉱物を思わせるごく弱い粒子
- 人物：原則として描かない。必要な場合は顔のないシルエット、背面、持ち物、足跡などへ変換する
- 象徴：各カードで「主象徴1つ＋補助象徴2〜4つ」までに絞る
- 空間：中央を埋め尽くさず、画面の25〜45%程度を静かな余白として残す

### Geometry quality gate

- 太陽・月・光輪・車輪・円環など、円であること自体が象徴となる形は正円にする。意図のない楕円化や輪郭の波打ちは許容しない
- 同心円は中心を一致させ、車輪の放射線は等角度、中心軸は垂直に揃える
- 柱・剣・塔・支柱など建築的な形は、遠近表現を意図した管所以外では水平・垂直・左右対称を確認する
- 五芒星、正方形、三角形などの幾何記号は、カードの意味に必要な向きと頂点数を保ち、生成時の歪みを残さない
- 植物、山、布、水面、炎、動物などの有機的な形は、機械的な対称化をせず自然な揺らぎを残す
- 書き出し後に22枚を一覧表示し、単体画像と縮小表示の両方で円・軸・枠・文字を再確認してから採用する

## 3. Color rules

| Role | Approximate color | Use |
| --- | --- | --- |
| Paper | `#F2EEE6` | 明るいカードの地 |
| Charcoal | `#242426` | 暗いカードの地、崖、影 |
| Grey mauve | `#A9A0AC` | 天体、水面、遠景 |
| Deep mauve | `#706875` | カード内の中間影 |
| Muted gold | `#AA8846` | 星、細線、焦点となる小さな光 |

1枚につき、主背景1色・主形状1色・補助色1色・金のアクセントを基本とする。金は面積を広げず、視線の焦点または象徴同士を結ぶ細線に限定する。

## 4. Typography

- カード番号：細身のセリフ体、中央揃え
- カード名：細身の大文字セリフ体、広い字間
- アプリUI：見出しはセリフ体、説明は端末標準のサンセリフ体
- 画像生成で文字を確定させず、最終制作では共通テンプレートで番号とカード名を合成して表記揺れを防ぐ

## 5. Abstraction policy

1. RWSの主要象徴と、その象徴が担う意味を先に整理する。
2. 意味を保つために不可欠な象徴を1つ選び、主象徴にする。
3. 人物の動作は、向き・距離・高低差・光の方向・残された物へ変換する。
4. 同じ図形を全カードへ機械的に反復せず、カード固有の象徴を優先する。
5. 抽象化後も、RWSを知る人が対応関係を説明できるか確認する。

## 7. Question workflow

TODAY'S QUESTIONは画像とは別に先行量産しない。各カードについて、RWSの意味を確認し、Deck 01で強調する視覚モチーフを決め、画像を完成させた後、その画像内の具体的な形・距離・光・余白を一つだけ拾って問いへ変換する。

正位置と逆位置は、同じ問いの肯定・否定にはしない。正位置では象徴が開く方向、逆位置では象徴が滞る・閉じる・過不足になる方向を見る。どちらも答えや行動を要求せず、短い一文で閉じる。

## 8. App icon policy

アプリアイコンは個別デッキの一部ではなく、すべてのデッキを収める共通の入口として扱う。深い黒紺の地に、大きな白い四芒星と、ごく淡いプリズムの粒子を置く。カード固有の象徴・番号・スート・デッキ色は使用せず、将来デッキが増えても原則として変更しない。

## 9. Minor Arcana system

小アルカナは大アルカナと同じ紙質・枠・余白・タイポグラフィを使い、スートだけで別デッキのように見えないようにする。色ではなく、素材と形によってスートを区別する。

| Suit | Core form | Supporting space | Geometry rule |
| --- | --- | --- | --- |
| WANDS | 黒い木、芽、細い火 | 丘、道、乾いた空間 | 本数と進行方向を明確にする |
| CUPS | 黒い器、金の流れ | 水面、波、月 | 器の直立・転倒・空隙を区別する |
| SWORDS | 黒い直線、金の柄 | 風、雲、水平線 | 刃の本数・角度・平行を正確にする |
| PENTACLES | 正円と正五芒星 | 土、庭、建築、植物 | 円を正円にし、星の五頂点を保つ |

数札は、記号を数だけ並べるのではなく、RWSの場面が担う関係を「距離・高低差・整列・欠落・移動」へ抽象化する。Courtは顔のない幾何学的な形で表し、PAGEは小さな淡色の形、KNIGHTは馬と斜線、QUEENは包む形と有機物、KINGは高い石の座と中心軸で区別する。

小アルカナの文字・枠・正円・直線は共通テンプレートで合成し、画像生成由来の表記揺れや歪みを持ち込まない。完成後はスート別一覧と78枚一覧の両方で、本数、カード名、中心軸、余白を確認する。

中央図像には、大アルカナと共通する紙繊維、鉱物顔料の濃淡、半透明の水彩層、浅い影を加える。これらは図形の外形を変形させずに重ね、杖・剣の本数、金貨の正円、五芒星、枠と文字の精度を維持する。暗色カードは内枠全体を一枚の暗色面として構成し、番号・図像・カード名欄を別々の黒い長方形に分断しない。

## 10. App UI — Pure MATTE

アプリUIはカードを展示する中立的な暗色空間として扱い、Deck 01のラベンダーや金をUI全体のブランド色にはしない。背景は青みをわずかに含む墨黒 `#0B0C0F`、主要Surfaceは `#16181D`、強調Surfaceは `#22252B`、本文は `#E6E6E6`、補助文は `#A1A1A8` を基準とする。

- 面は不透明なマット。Glassmorphism、背景ぼかし、ネオン、広いグラデーションは使用しない
- 白光明朝は「今日の一枚」、カード名、HISTORY / SETTINGS、今日の問いなど短い見出しに限定する
- 本文、ボタン、日付、補助情報は端末の読みやすい日本語サンセリフを使う
- プリズム✦はTODAY上部とアプリアイコンだけに使い、通常の区切りや選択状態はオフホワイトの小さな✦で表す
- カード周辺へUI由来の豪華な枠や光彩を追加しない
- Bottom Navigationは不透明な暗色面とし、選択状態は明度・線・小さな点で示す
